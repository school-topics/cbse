## What this tells us

The app's **core is solid** (families/ℕℤℚℝ, fractions, distance, decimal
expansions). The book goes deeper in a well-defined set of **[gap] interactives**,
all living on the number line:

**Deep-dive backlog (proposed, easy → hard):**
1. **Rational on the number line** (C3-4.3) — pick q, split the unit interval into q parts, step p.
2. **Cyclic number 142857 shifter** (C3-6.2) — multiply by 1–6, watch the digits rotate.
3. **Terminating-or-repeating predictor** (C3-6.2 / 6.1) — factor the denominator; predict, then verify by long division.
4. **Density: average between two** (C3-4.4) — insert (a+b)/2 repeatedly; infinitely many.
5. **Decimal → fraction converter** (C3-6.1) — the 10ⁿx − x algebra, staged like Examples 5–9.
6. **√n square-root spiral** (C3-5.3 / EOC-16) — construct √2, √3, … and read each hypotenuse.
7. **Integer fortunes/debts jumps** (C3-3.2) — add/subtract integers as jumps on the number line; Brahmagupta's sign rules made visible.

Plus a **[partial]→tighten** pass: the √2-from-unit-square visual (C3-5.1).
Proofs (√2, √5 irrational;
0.999…=1) stay as **reading/reasoning** (step-through), not forced interactives.

**Score (current app, est.):** ~9 covered · ~9 partial · ~6 gap · ~8 text.
Target: close the 6 gaps as deep-dives, tighten the partials via the question
bank + section shell, keep text as reading.
