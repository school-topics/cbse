# Deep-dive spec — Vertices from midpoints (C1-3.7)

**Coverage-map ID:** `C1-3.7` (End-of-Chapter Q13)
**Tier:** HARDEST — the reverse problem. Top of the Challenge ladder.
**Concept:** given the three midpoints of a triangle's sides, reconstruct the
three vertices. Uses vertex = (sum of two adjacent midpoints) − (opposite one).

## Why this exists
Book Q13: the midpoints of the sides of triangle ABC are D, E, F. Find A, B, C.
This is a genuine reverse problem — most students find it hard, which is exactly
why it belongs at the top of the ladder as a capstone that rewards mastering
midpoints (C1-3.5) first.

## Behaviour (must-haves)
1. Show three fixed midpoint markers D, E, F (start with the book values
   D(5,1), E(6,5), F(0,3)).
2. Three draggable candidate vertices A, B, C the student positions.
3. Live feedback: for the current A,B,C, compute the midpoints of AB, BC, CA and
   show whether each lands on D, E, or F. Highlight matches.
4. Success when all three computed midpoints coincide with D, E, F (tolerance).

## Discovery scaffold (recommended)
- Stage 1: let them try freely (they'll struggle — that's fine).
- Stage 2: hint — "the midpoint of two vertices is their average; can you work
  backwards?" Show one relationship, e.g. midpoint(A,B)=D means A+B=2D.
- Stage 3: reveal the trick: A = D + F − E, B = D + E − F, C = E + F − D, with a
  one-line why (add the three "sum" equations, solve).

## Nice-to-haves
- A "solve it for me, step by step" button that animates the three vertices
  snapping into place with the arithmetic shown — for after they've tried.
- Let them change D,E,F and see the triangle regenerate, reinforcing that the
  relationship is general.

## Correctness checklist
- Formula: A = D+F−E, B = D+E−F, C = E+F−D (D=mid AB, E=mid BC, F=mid CA).
- Book values D(5,1),E(6,5),F(0,3): A = (5+0−6, 1+3−5) = (−1,−1);
  B = (5+6−0, 1+5−3) = (11,3); C = (6+0−5, 5+3−1) = (1,7).
- Verify by forward-check: mid(A,B) = ((−1+11)/2,(−1+3)/2) = (5,1) = D ✓;
  mid(B,C) = ((11+1)/2,(3+7)/2) = (6,5) = E ✓; mid(C,A) = ((1−1)/2,(7−1)/2)
  = (0,3) = F ✓. The app MUST pass this forward-check.
- Watch the labelling convention (which midpoint belongs to which side). Fix it
  once, document it, and keep it consistent with C1-3.5.

## Reference note
Hardest to get right because of the labelling convention and the reverse
arithmetic. The forward-check above is your ground truth — if the built activity
reproduces A,B,C from the book's D,E,F, it's correct. Test that first, before
any polish.
