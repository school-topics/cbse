# Deep-dive spec — Collinearity without plotting (C1-3.3)

**Coverage-map ID:** `C1-3.3` (End-of-Chapter Q6–Q7)
**Tier:** medium (live computation as points move)
**Concept:** three points are collinear iff the slopes between them are equal
(equivalently, the "triangle" they form has zero area). Discover a test that
needs no plotting.

## Why this exists
Book Q6–Q7 ask students to decide if three points lie on a straight line
*without plotting*, then verify by plotting. The app should let them build the
test themselves, then see it confirmed visually.

## Behaviour (must-haves)
1. Plane with three draggable points A, B, C.
2. Live readouts as they drag:
   - slope AB = (yB−yA)/(xB−xA) and slope BC = (yC−yB)/(xC−xB), shown as
     fractions where possible.
   - a verdict line: "collinear" when the two slopes are equal (tolerance),
     else "not collinear."
3. Draw the segment through the points; when collinear, A, B, C visibly line up.

## Nice-to-haves
- Second method toggle: the "area = 0" test using
  area = ½|xA(yB−yC)+xB(yC−yA)+xC(yA−yB)|; show area shrinking to 0 as they
  align the points. Two methods, one truth — reinforces the concept.
- "Load book points" buttons: M(−3,−4),A(0,0),G(6,8) → collinear (slope 4/3);
  R(−5,−1),B(−2,−5),C(4,−12) → NOT collinear (let them discover).

## Integration
- New deep-dive under Coordinates. `data-ncert="C1-3.3"`.
- Reuse axis helper. Slope display needs a small fraction-formatter.

## Correctness / pedagogy checklist
- Vertical lines: xB−xA = 0 makes slope undefined. Handle it — if all three
  share the same x, they're collinear (vertical line); don't divide by zero.
  This is the main correctness trap.
- Tolerance for "equal slopes": compare cross-multiplied form
  (yB−yA)(xC−xB) == (yC−yB)(xB−xA) to avoid division entirely — cleaner and
  dodges the vertical-line problem. Recommend this over literal slope division.
- Book verify: M,A,G give (−4−0)(6−0)=(8−0)(0−(−3))? → (−4)(6)=−24 vs (8)(3)=24
  — wait, order matters; use consistent point order. Document the exact formula
  you implement and test it on both book triples before shipping.

## Reference note
The cross-multiplication form is both more correct (no vertical-line crash) and
better pedagogy (integer arithmetic students can verify by hand). Prefer it.
This is where a review pass earns its keep — get the formula and point-order
right once.
