# Deep-dive spec — Pixel screen (C1-3.9)

**Coverage-map ID:** `C1-3.9` (End-of-Chapter Q15)
**Tier:** easy
**Concept:** coordinates in computing; origin at bottom-left; testing whether a
shape fits on screen and whether two circles overlap, using the distance idea.

## Why this exists
Book Q15: an 800×600 screen with origin bottom-left. Circle icon A (r=80) at
(100,150); circle B (r=100) at (250,230). Do any parts lie off-screen? Do the
circles intersect? Bridges coordinates to why *this very app* uses a coordinate
system — a strong motivator.

## Behaviour (must-haves)
1. Draw a rectangle representing the screen (scaled to fit the widget), with the
   origin marked at bottom-left and axes labelled. Show the true pixel scale in
   labels (0–800 x, 0–600 y) even though it's scaled down visually.
2. Two draggable circles A and B with editable centre + radius (sliders or drag).
3. Live readouts:
   - For each circle: does any part cross an edge? (centre.x − r < 0, or
     centre.x + r > 800, etc.) Flag which edge.
   - Do A and B intersect? Rule: they intersect iff distance(centreA,centreB)
     < rA + rB (and > |rA − rB| for "not one-inside-other"; keep it to the
     simple overlap test for grade 9: distance < rA+rB → overlap).
4. Show the distance between the two centres numerically as the student drags.

## Nice-to-haves
- "Snap to book values" button that sets A=(100,150,r80), B=(250,230,r100) so
  the student can reproduce the exact textbook problem and check the answer.
- Toggle origin to top-left to show how screen coordinates sometimes flip y
  (common in real graphics) — a nice "did you know."

## Correctness checklist
- Off-screen test uses the *pixel* coordinates (0..800, 0..600), not the scaled
  drawing coordinates. Keep model coords and screen coords separate.
- Overlap test: for the book values, distance = √((250−100)²+(230−150)²)
  = √(22500+6400) = √28900 = 170. rA+rB = 180. 170 < 180 → they DO intersect.
  Verify the app reports this.
- Circle A: 100−80=20 ≥0 and 150−80=70 ≥0 and 100+80=180 ≤800 and 150+80=230
  ≤600 → fully on screen. Circle B similar → fully on screen. Verify.
