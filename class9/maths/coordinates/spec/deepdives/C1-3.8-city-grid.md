# Deep-dive spec — City street grid (C1-3.8)

**Chapter:** Ganita Manjari G9, Ch.1 Coordinates
**Coverage-map ID:** `C1-3.8` (End-of-Chapter Q14)
**Tier:** easy (self-contained, one concept, ~one session)
**Concept taught:** an ordered pair names one unique point; order matters,
so (a,b) ≠ (b,a). Real-world model = city street intersections.

## Why this exists
The book (Q14) models a city as N–S and E–W streets and names each
intersection like (2,5). It asks how many crossings are "(4,3)" vs "(3,4)".
Current app has no activity for this. It also loops back to the §1.1 history
(Sindhu-Sarasvati grid) and forward to Q15 (pixel screen).

## Behaviour (must-haves)
1. Render an N×N grid of streets (start N=6; the book uses 10 — make N a const
   so it's trivial to raise). Two axes emphasised (the two main roads).
2. Each intersection is a tappable node. Tapping selects it and displays its
   name as `(nsStreet, ewStreet)`, mono font, prominent.
3. Selecting a new node deselects the previous one (single selection).
4. Two guided questions as buttons, resolving the book's puzzle:
   - "How many crossings are (4,3)?" → exactly one (a name fixes both streets).
   - "Why is (4,3) ≠ (3,4)?" → order matters; swapping gives a different crossing.
5. Touch + mouse both work. Nodes have a ≥44px effective tap target (use a
   transparent hit-circle r≈9 over a small visible dot).

## Nice-to-haves (later)
- A "highlight (4,3)" mode that flashes the single matching crossing, and a
  "highlight all with N–S=4" mode that lights a whole street, making
  "one crossing vs a whole line" visceral.
- Distance between two selected crossings (200 m per step) — bridges to §1.4.

## Correctness checklist (for your review pass)
- (a,b) with a≠b must select a visibly different node than (b,a).
- Node count = N×N; corner main-road axes are the (0,*) and (*,0) lines.
- The "how many are (4,3)" answer is 1, not "several" — a common student slip
  the activity should actively correct.

## Reference implementation
A working prototype (HTML+SVG+vanilla JS, no deps) was built in chat — same
grid, naming, and two-question logic described above. Port that into the app
shell rather than starting from scratch; swap inline colors for your app's
tokens and the transparent/dot hit-target pattern is already correct.
